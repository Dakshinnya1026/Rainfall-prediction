# 🌧️ Rainfall Prediction App

An end-to-end machine learning web app that predicts **next-day rainfall**
across Australia, trained on 10 years of real weather station data from the
Australian Bureau of Meteorology.

**Live demo:** _add your deployed URL here after following the steps below_

## Features

- **3 models compared** — Logistic Regression, Random Forest, XGBoost —
  evaluated on accuracy, precision, recall, F1, and ROC-AUC. Best model
  selected automatically.
- **Class-imbalance handling** — rain days are the minority class (~22%),
  so models use class weighting instead of naive accuracy optimization.
- **Live weather auto-fill** — pulls real current conditions for any city
  via the free Open-Meteo API to pre-fill the prediction form.
- **Model transparency dashboard** — ROC curves, confusion matrix, and
  feature importances, not just a single prediction.
- **Data explorer** — interactive charts on seasonality, humidity vs. rain,
  rainiest locations, and feature correlations.

## Run locally

```bash
pip install -r requirements.txt
python train.py        # trains models, saves best pipeline to models/
streamlit run app.py   # launches the web app at http://localhost:8501
```

## Deploy for free (Streamlit Community Cloud)

1. Push this folder to a new **public GitHub repo**.
2. Go to https://share.streamlit.io and sign in with GitHub.
3. Click **"New app"**, select your repo, branch `main`, and main file `app.py`.
4. Click **Deploy**. You'll get a live URL like
   `https://your-app-name.streamlit.app` — put this on your resume/portfolio.

> The `models/` folder (trained pipeline + JSON metadata) must be committed
> to the repo since Streamlit Cloud doesn't run `train.py` for you — it just
> runs `app.py`.

## Dataset

Source: [Rain in Australia](https://www.kaggle.com/datasets/jsphyg/weather-dataset-rattle-package)
— ~145,000 daily observations (2008–2017) from 49 Australian weather
stations, originally published by the Bureau of Meteorology.
